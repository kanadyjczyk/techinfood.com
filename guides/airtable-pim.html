Setting Up Airtable
as a PIM for Food Businesses

Free guide Intermediate ⏱ 15 min read Updated 2025

Most small food businesses manage product data across spreadsheets, their website backend, a price list, and a handful of email threads. When something changes — a new allergen declaration, a price update, a new image — someone has to update it in four different places. And someone always misses one.

A PIM (Product Information Management) system fixes this. One place for all product data. Every channel pulls from it. Change once, update everywhere.

Enterprise PIM software costs tens of thousands per year. For most small food businesses, Airtable does the job at a fraction of the cost — and your team can use it without training.

What this guide covers: Setting up your Airtable base, adding all required food product fields including allergens and nutritional data, linking suppliers and categories, attaching media, and connecting to Shopify via REST API.

Step 1 — Create your Airtable base

Step 01

Create a new base called "Product Catalogue"

Sign up at airtable.com (free plan supports up to 1,000 records — enough for most small food businesses). Create a new Base and name it Product Catalogue. Inside it, you'll have your first table automatically — rename it Products.

Step 3 — Add allergen and nutritional fields

This is where Airtable really earns its place for food businesses. The UK requires declaration of 14 major allergens. Add a Multiple select field called Allergens with all 14 as options:

Step 03

Create Categories and Suppliers tables

Add two more tables to your base: Categories (fields: Name, Description) and Suppliers (fields: Name, Contact name, Email, Phone, Lead time days, Payment terms). Then go back to your Products table and change the Category and Supplier fields to Link to another record pointing at these tables.

Step 4 — Connect to Shopify via REST API

Once your Airtable PIM is populated, you'll want product data to flow automatically to your Shopify store. There are two approaches:

Option A — Make.com (recommended for non-developers): Create a Make.com scenario that watches your Airtable Products table for changes. When a record is updated, it calls the Shopify REST API to update the matching product. No code required.

Option B — Direct API call: If you have developer access, you can call both the Airtable API and Shopify API directly.

// Airtable API — fetch a product record GET https://api.airtable.com/v0/{baseId}/Products/{recordId} Authorization: Bearer YOUR_AIRTABLE_TOKEN // Shopify REST API — update a product PUT https://{shop}.myshopify.com/admin/api/2024-01/products/{id}.json X-Shopify-Access-Token: YOUR_SHOPIFY_TOKEN Content-Type: application/json { "product": { "title": "Organic Oat Milk 1L", "body_html": "Your product description", "variants": [{"price": "2.99", "sku": "OOM-001"}] } }
Need help with the API connection? This is the part most small food businesses ask for help with. or walk through it with you on a call.

Tips for keeping your PIM accurate

  • Make one person responsible for product data. Assign a "Product Data Owner" — usually someone in buying, marketing, or operations.
  • Use Airtable forms to collect data from suppliers. Send them a form link to fill in allergens and nutritional values — the responses go straight into your base.
  • Add a "Last updated" date field and a "Data complete?" checkbox. Filter for incomplete records and review them monthly.
  • Set up Airtable automations to send a reminder email when a product reaches its review date or when allergen fields are empty on a Live product.
